# Social Cards System

The social cards system for contributor.info provides rich, dynamic preview images for social media sharing. When users share links to the site on platforms like Twitter, Facebook, LinkedIn, or Discord, these cards enhance the visual presentation with repository statistics, contributor data, and branding.

## Architecture Overview

### Dynamic Generation via Fly.io
Social cards are now dynamically generated using:
- **Fly.io Service** for real-time card generation
- **Supabase Integration** for live data
- **SVG Format** for lightweight, scalable images
- **CDN Caching** for optimal performance

### Service Endpoints
```
https://contributor-info-social-cards.fly.dev/
├── /health                          # Service health check
├── /metrics                         # Performance metrics
├── /social-cards/home              # Homepage social card
├── /social-cards/repo?owner=X&repo=Y  # Repository cards
└── /social-cards/user?username=X   # User profile cards
```

## Development Workflow

### Testing Social Cards
```bash
# View the preview interface
npm run dev
# Navigate to http://localhost:5173/dev/social-cards

# Test the Fly.io service directly
curl https://contributor-info-social-cards.fly.dev/health

# Generate a test card
curl "https://contributor-info-social-cards.fly.dev/social-cards/repo?owner=facebook&repo=react"
```

### Testing Cards
```bash
# Test meta tags and platform compatibility
npm run test-social-cards

# Monitor CDN performance
npm run monitor-cdn

# Visual regression testing
npm run chromatic
```

### Adding New Cards
1. **Update Fly.io service** in `fly-social-cards/src/card-generator.js`
2. **Add new endpoint** in `fly-social-cards/src/server.js`
3. **Update meta tags** in `src/components/common/layout/meta-tags-provider.tsx`
4. **Create Storybook story** for visual testing
5. **Deploy to Fly.io** and test across platforms

## Troubleshooting

### Common Issues

**Cards not generating**
- Check `SUPABASE_TOKEN` environment variable
- Verify Supabase Storage bucket exists
- Ensure dev server is running during generation

**Visual inconsistencies**
- Check font loading in card components
- Verify consistent mock data in stories
- Test across different browsers/platforms

**CDN issues**
- Monitor cache hit rates with `npm run monitor-cdn`
- Check Supabase Storage dashboard for file status
- Verify public URL accessibility

### Platform Validation
Use these tools to validate social cards:
- **Twitter**: https://cards-dev.twitter.com/validator
- **Facebook**: https://developers.facebook.com/tools/debug/
- **LinkedIn**: https://www.linkedin.com/post-inspector/
